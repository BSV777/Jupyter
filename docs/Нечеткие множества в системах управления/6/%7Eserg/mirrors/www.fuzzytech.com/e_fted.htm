Open Software Interfaces

fuzzyTECH can be linked and integrated with most other software products. We have tried to support both most industry standard interfaces (DLL, OLE, ANSI-C, ...), as well as specific "plug-and-play" type interfaces for frequently used products (Simulink, InTouch, Excel, VisualBasic, ..) that provide an even tighter integration. Please contact us if you do not find the link to the software or hardware products you use on these pages. Here are some examples of fuzzyTECH's open software interfaces:

  • MS-Windows Interfaces: DDE, DLL (16 and 32 bit), OLE/ActiveX.
  • fuzzyTECH uses FTL (Fuzzy Technology Language) to store its projects on disk.
  • Built-in M code generator for control systems simulation software such as VisSim™ and Matlab/SIMULINK™.
  • For MS-Excel, fuzzyTECH comes with the "fuzzyTECH Assistant", an Excel-Add-In that you install within MS-Excel which allows you to link cells and arrays of cells to fuzzy logic functions.
  • A VisualBasic source module lets you tie in fuzzy logic functions and even the complete editors and analyzers of fuzzyTECH by "point-and-click".
  • Dedicated fuzzyTECH's Runtime Modules are available for Programmable Logic Controllers (Siemens, Klöckner-Moeller, Allen-Bradley, InControl, ..) and Process Control Systems (InTouch, TheFIX, FactoryLink, WinCC, Foxboro-IAS, DIGIMATIK, Freelance2000, Contronic, Citect, LabVIEW..)

Customized for Your Application

fuzzyTECH comes with an Application Programming Interface that allows you to remotely control most functions in fuzzyTECH from your software. This API is provided as a DLL called FTRCU (fuzzyTECH Remote Control Unit) which contains all the functions you need to completely integrate fuzzyTECH with your program.
