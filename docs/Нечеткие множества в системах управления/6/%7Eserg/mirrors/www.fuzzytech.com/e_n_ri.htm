fuzzyTECH Release Information

This page contains general information about:
1. new functions added with the latest releases,
2. fuzzyTECH's Release Policy, and
3. Maintenance Updates.

1. New Functions in fuzzyTECH

New Functions in Release 5.1 over Release 5.0:

  1. Release 5.1 features a major redesign of its user interface. The LV window (list of all linguistic variables) has been replaced by a powerful "treeview" window. This treeview displays a complete fuzzyTECH project as a structure similar to the Windows Explorer featured with MS-Windows'95 and /NT 4.0.
  2. Throughout fuzzyTECH, many dialogs have been restructured and merged with others into tabbed dialogs for easier access.
  3. The DDE link capabilities of fuzzyTECH have been modified to a simple "cut&paste" approach similar to MS-Office software products, such as MS-Excel. For an example, see the screenshot.
  4. The Trace function has been redesigned and its features have been extended. You may now conduct traces both remotely on the target hardware as well as locally on the PC.
  5. The Pattern Generator has been redesigned for easier and quicker use.
  6. The Debug window has been extended and enhanced. It is now called "Watch" window.
  7. New setup routine using InstallShield: We have created a completely new setup routine that lets you simultaneously install multiple Editions of fuzzyTECH into the same directory on your PC in one step.
  8. In fuzzyTECH, you may now switch "life" between all the Editions of fuzzyTECH you have installed. Simply select the Edition you want to work with from the Edition option of the Tools menu.
  9. You may now install any Edition of fuzzyTECH as Demo version. Thus, there is no dedicated "Demo" of fuzzyTECH anymore; every fuzzyTECH product falls back into Demo mode, if you do not have the license for it.

New Functions in Release 5.0 over Release 4.2:

  1. Release 5.0 now is a complete 32-bit software program and runs only under MS-Windows95 and MS-Windows/NT 4.0. We will continue to deliver and support release 4.2 for use with 16 bit operating systems. A demo of the release 5.0 is available for download.
  2. New look&feel: Release 5.0 supports the new elements of the 32-bit Windows API such as long file names, tabbed dialog boxes, colored toolbars, ToolTips, etc..
  3. Support of new target hardware platforms: The fuzzyTECH IA-S5 Edition has been renamed to fuzzyTECH IA-S5/7 Edition and now supports the SIMATIC™ S7 family of PLCs FOR THE SAME PRICE. The new fuzzyTECH MCU-HC11/12 Edition, developed in cooperation with Motorola, now supports both HC11 and HC12 family microcontrollers. A fuzzyTECH MCU-HC05/08 Edition for the 68HC05xx and 68HC08xx families is currently under development.
  4. Free choice of communication drivers for the online communication between fuzzyTECH and the target hardware platform. In addition to the serial port and the shared file system (network), we now allow the integration of user-defined communications DLLs (for example for CAN bus or industrial automation field busses).
  5. The "fuzzyTECH Precompiler Edition" is now named "fuzzyTECH Professional Edition" because of the functionality added with the release 5.0. It now contains the new stand-alone runtime modules for process control systems (InTouch, InControl, WinCC, CiTECT, LabVIEW, ..) and the runtime DLLs (FTRUN32.DLL,..). These are also provided with the fuzzyTECH Online Edition.
  6. Release 5.0 provides the HyperInference technology as an add-on module. This new and innovative fuzzy inference technology has been invented and patended by Prof. Kiend and exclusively licensed for fuzzyTECH by Inform. The fuzzyTECH Demo 5.0 already contains the new HyperInference module.

New Functions in Release 4.2 over Release 4.1:

  1. New Documentation Generator that creates complete project description at the push of a button.
  2. Support for new target hardware platforms: Intel MCS®-251 microcontroller family and Siemens SIMATIC™ S5 programmable logic controllers.
  3. New 32-bit interfaces added. Version 4.2 now supports interfacing with both 16-bit software and 32-bit software. For example, you can now tie in fuzzy logic systems with both 16-bit MS-Excel 5.0 and 32-bit MS-Excel 7.0…
  4. New Rule Wizard and Variables Wizard expedite development even further.
  5. New Stand-Alone fuzzyTECH Runtime Module for fuzzyTECH Online and Precompiler Edition, as well as fuzzyTECH for Business PlusC. The Runtime Module can be integrated with any other software and processes *.FTR-files generated by fuzzyTECH as stand-alone modules. The currently available incarnations of the Runtime Module are 16-bit and 32-bit dynamic link libraries (DLL). No royalties or additional licensing applies when you distribute the Runtime Module with your software or systems solution. To visualize the running fuzzy logic system, the fuzzyTECH Online Edition in addition contains the "fuzzyTECH Viewer", a limited version of fuzzyTECH that can be distributed royalty-free with the Runtime Module.
  6. Misc. enhancements: The linguistic variables editor now shows a tool-tip style pop-up window with the coordinates of a highlighted membership function definition point. Rule Blocks and Interfaces in the Project Editor, as well as Linguistic Variables in the Variables list box, display any developer comments in a tool-tip style pop-up window when the respective window is activated and the mouse pointer rests over the respective object. When you start fuzzyTECH for the first time, a splash screen gives you access to the different types of online documentation of fuzzyTECH. A new one-page, the "QuickHelp" page, has been added. When you edit anything in an opened FTL project, a "*" character in the title line of fuzzyTECH appears.

Release 4.1 Improvements Over Release 4.0:

  1. The Online Edition supports online communication via common file system in addition to the serial communication. This allows for a target system and fuzzyTECH to do online communication over most network systems.
  2. The NeuroFuzzy Module now comes with a FuzzyCluster add-on Module. The FuzzyCluster Module supports both FuzzyClustering and IsodataClustering. To start clustering, select the option in the "Neuro" menu of fuzzyTECH. For a complete description of the functionality, refer to the FuzzyCluster section of the online help system that you open by selecting the "FuzzyCluster" option of the "Help" menu of fuzzyTECH. Clustering is primarily used to pre-process training data for the NeuroFuzzy Module.
  3. You can specify your own editor to be called by fuzzyTECH each time ASCII data is displayed or edited in the "Preferences" dialog that opens when you select "Preferences..." from the "Options" menu of fuzzyTECH.
  4. The Online Edition supports the new interface types "Visualization" and "Force". These interfaces let you design complete man-machine interfaces (MMI) on top of fuzzyTECH using programming languages such as VisualBasic or Visual C++. Refer to the online help system of fuzzyTECH for more details.
  5. You can switch the language of the menus and dialogs (currently English and German are supported). Select "Languages.." from the "Options" menu of fuzzyTECH. A German language manual and online help system will be available in a later release.
  6. A Revision Control System (RCS) has been added. Open the Revision Control dialog by selecting "Revision Control" from the "File.." menu of fuzzyTECH. A RCS lets you document and save each stage of development of a fuzzy logic system. Refer to the online help system of fuzzyTECH for more details.
  7. The fuzzyTECH MP Edition is expanded to support all new PIC microcontrollers. A special library for the PIC17CXX devices providing a hardware-multiply is added. 16-bit computational resolution, banked memory support, and weighted rules are now available for most parts. Please read the specific README.TXT file of this Edition.
  8. A new "Attributes" button has been added to the main toolbar. Depending on the object currently highlighted, the respective dialog is opened.
  9. A "Close" option has been added to the "File" menu of fuzzyTECH.
  10. The "Anyfile" type (*.*) has been added as Filetype in all File Selector Boxes.
  11. Release 4.1 uses a new Setup routine that lets you specify individual components for installation. For example, you now can choose not to install the sample files.

2. fuzzyTECH Release Policy

The release number of every software product of the fuzzyTECH family contains three digits (for example: 4.23). The first digit, left of the decimal point, is the major release number (generation number). A change to this first digit indicates that the sofware has undergone significant modifications which affect most components. Major releases are issued at approximately one to two year intervals. The first digit to the right of the decimal point is the minor version number. A change to this number indicates that minor modifications or enhancements have been made to the software. Typical examples for such minor changes include new components or additional target platform support. In other words, changes that do not affect a great number of the software components. A change to the second digit to the right of the decimal point indicates a maintenance release. This number is changed each time a new build to the entire product range is made. These changes are typically not visible to the user, involving bug fixes and resolving conflicts with other software/hardware components (although we try hard to deliver highly stable software products and invest heavily in quality control, the state of the art in software design does not allow for 100%-never-fail-software). Because maintenance releases do not involve changes in functionality, the fuzzyTECH release numbers cited in most of our documentation refers only to the major release and minor version numbers.

If you have a software update contract, you are automatically sent a new release of your products for every major and minor release as they become available. Major release updates include a new set of manuals while minor releases come with short descriptions of the new features and modifications. If you purchase software products or updates from INFORM, you will always get the latest available release with the latest available documentation. Note, the release numbers of all add-on modules (NeuroFuzzy, FuzzyCluster, RTRCD, ...) must match those of the fuzzyTECH Edition with which they are supposed to run. Hence, add-on modules must always be updated with the respective fuzzyTECH Edition.

3. Maintenance Updates

We now make updates to new maintenance releases available for free to every fuzzyTECH user. For example, if you have licensed fuzzyTECH Professional release 5.1, then you are entitled to receive a free update to the latest manitenance release available. For example, if we have delivered release 5.10 to you, you may download and install maintenance releases 5.11, 5.12, 5.13, ...

To check the release number of the latest available maintenance release, and to download&install it, please refer to the fuzzyTECH Installation page. Notice that for technical reasons, maintenance releases are only available from this web sites for releases later than 5.10.
