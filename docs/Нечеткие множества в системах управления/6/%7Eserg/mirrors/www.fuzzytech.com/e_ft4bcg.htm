fuzzyTECH for Business / Back-Ends

There are two alternatives for using a completed fuzzy logic system. The first is to use fuzzyTECH directly to process data or to tied in with other standard software packages, such as spreadsheet and database software. The second alternative is to use one of the ready-to-use back-ends of fuzzyTECH.

1. Using fuzzyTECH to Process Data

Once you have developed a fuzzy logic solution within fuzzyTECH, you can use it in a number of ways:

  • Enter the input data manually into fuzzyTECH in Interactive debug mode and observe the fuzzy logic inference and its results in fuzzyTECH's Editors.
  • Process data files automatically using either File Recorder Mode or Batch Mode.
  • Integrate fuzzyTECH with MS-Excel using the MS-Excel fuzzyTECH Assistant, with MS-Access/VisualBasic using the provided Basic Source Code module, or with any other MS-Windows software throough fuzzyTECH's standard DDE/DLL interfaces.
  • Integrate fuzzyTECH completely with your software using the fuzzyTECH API (FTRCU.DLL). This is for experienced programmers only.

The advantages of this are:

  • You can interactively follow the entire inference process in all editors and analyzers of fuzzyTECH since these integration alternatives always integrate the complete fuzzyTECH software.
  • You can modify the fuzzy logic system on-the-fly using all fuzzyTECH editors.

However, there are some cases when you cannot use fuzzyTECH to process the data. These include:

  • You need better performance. On a typical PC, you can process up to 10 data sets per second in fuzzyTECH. In some applications you need to process many thousands of data sets per second.
  • You need to run the fuzzy logic system on a database server, a mainframe computer, or in a middleware environment such as SAP R/2 and R/3.
  • You want to distribute your designed fuzzy logic system to users that should not be able to re-engineer or modify the system.
  • You want to distribute a royalty-free, run-time version of your fuzzy logic system.

2. Using fuzzyTECH's Back-Ends

For these applications, INFORM has created fuzzyTECH for Business Plus-C and Plus-COBOL. You can purchase these versions later as an upgrade to your fuzzyTECH for Business:

fuzzyTECH for Business PlusC:

In addition to fuzzyTECH for Business, this version supports PC, workstation, and client/server type deployment of stand-alone fuzzy logic systems. For PCs, a stand-alone fuzzy logic runtime module (FTRUN.DLL) is provided both in 16-bit and 32-bit versions. You may link this module to any of your applications royalty-free. The stand-alone runtime module is very small (50KB to 100KB) and very fast (up to 10.000 data sets per second for a large fuzzy logic system on a Pentium-type PC). fuzzyTECH for Business Plus-C stores a fuzzy logic system as *.FTR files, a compressed and encrypted version of the *.FTL file. The fuzzy logic runtime module uses the *.FTR files rather than the *.FTL files. Note, whenever you change the fuzzy logic system, you only need to replace the respective *.FTR file on any PC that uses your fuzzy logic system. You do not need to change the fuzzy logic runtime module or your code.

For workstations and client/server systems, fuzzyTECH for Business Plus-C creates a fuzzy logic system as royalty-free C source code. This C code can be compiled and linked with your applications to provide the fuzzy logic functionality to your software project.

fuzzyTECH for Business Plus-COBOL:

fuzzyTECH for Business Plus-COBOL enables the generation of COBOL 85 source code that can be used on most mainframe computers. The compiled code achieves outstanding performance. Mainframe computers compute up to 175.000 data sets per second.

Special Add-On Modules:

The PlusC and PlusCOBOL versions let you deploy your fuzzy logic solutions for all common hardware platforms used with business and financial applications. However, you must still provide the interface code that gets the values of the input variables of the fuzzy logic system and delivers the values of the output variables. For some typical business environments, INFORM provides dedicated "plug-in" type code modules that deliver this interfacing (SAP R/2&R/3, ORACLE, DB2, …). These solutions are provided by INFORM as consulting services. Please contact us for availability and a detailed quotation.
