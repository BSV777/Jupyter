Фази оценка риска при лизинге автомобилей

 Автоматическая система оценки риска при сдаче автомобилей в лизинг была создана BMW Bank GmbH из Германии и компанией Inform Software GmbH с использованием фази логики. Основной задачей банка было передать процесс оценки риска от банка  непосредственно дилеру автомобилей. Это позволяет ему оперативно принимать решение о заключении контракта.
 
  Система поддержки решения

Система оценки риска была объединена с программным обеспечением  РС которые использовалось дилерами при заключении контракта.  Эти РС могут быть связаны с Германской базой данных кредитных историй для получения данных о клиенте. Фази система состоит из трех различных модулей каждый из которых применяется для определенного типа клиентов частный, наемный или корпоративный. Для частных клиентов система учитывает такие факторы как возраст, семейное положение, время проживания по одному адресу.  Подобная расчетная плата  уже существовала и банк считал ее достаточно удачной для включения в фази систему.

 Частные клиенты

Рис. 1 показывает структуру системы оценки риска для частных клиентов.
Входная переменная "ScoreCard" является результатом оценки расчетной платы. Результат расчета используется с другими входными переменными "Unemploy" -«Не занятость»  и "Demographic" - «Демография» в левом блоке правил для вычисления риска"CustProfile".  Переменная "Unemploy" поступает из базы данных , которая хранит текущий уровень безработицы для профессии клиента. Переменная  "Demographic"  поступает из базы данных и характеризует уровень неликвидности для жилища клиента.
Результат оценки блока "CustProfile" служит входной переменной для блока который вычисляет рейтинг риска для лизингового контракта. В дополнение  к "CustProfile" блок правил использует входные переменные "Backpaying," "CredHist," and. "Ratio ". Переменная "Backpaying" описывает как клиент соблюдал предыдущие лизинговые контракты. Переменная "CredHist" поступает из Германской базы данных кредитных историй и описывает историю всех предыдущих  кредитов клиента  во всех банках. Переменная "Ratio" вычисляется как отношение ежемесячной платы к ежемесячному доходу клиента.

Корпоративные клиенты

Рис.2 показывает структуру системы оценки риска для корпоративных клиентов.
 Здесь процесс принятие решения более сложен и включает больше переменных. Самый правый блок вычисляет выходную переменную "Rating"(Рейтинг) на основании четырех выходных переменных  "IndustryType," "IlliquidRisk,","CredEval," and "Backpaying".  Переменная "IndustryType" поступает из базы данных  и описывает риски связанные с определенными видами  бизнеса в субъективной оценке банка. Переменная "Backpaying"  имеет тот же смысл, что и для частных клиентов.
 Две другие входные переменные поступают из другого блока. CredEval комбинация входных переменных "IndexCred" и  "InforCred". Оба поступают из информационной службы, которая поддерживает сведения о кредитной рейтинге каждой немецкой компании. "IlliquidRisk" оценка фази системой степени риска для определенной компании стать неликвидной. Блок правил вычисляет эту переменную из оценки финансового положения компании "FinanBack" и ее структуры "CompStruct". Финансовая оценка компании учитывает ее капитал ("Capital") и ее  годовой доход ("Revenue"). Структура компании оценивается исходя из числа сотрудников ("Workforce"), возраста компании ("Age") и юридического статуса ("LegalType").

 Результаты использования фази системы в BMW банке

 Общая фази система включала 413 логических правил в трех модулях. Полное проектирование , отладка и проверка системы потребовали два человека года. Интегрирование программного модуля в формате DLL сгенерированного при помощи пакета fuzzyTECH в программное обеспечение для заключения лизинговых контрактов потребовало человеко-месяц. Система в настоящее время используется при операциях BMW  дилерами и банк считает, что ее действия эквивалентны действиям опытного инспектора по лизинговым контрактам. Хотя расчет эффективности использования системы не опубликован банком, примерная оценка, основанная на том, что в год заключается 50000 лизинговых контрактов и затрачивается примерно 30 минут на оценку каждого контракта (включая получение информации о кредитной истории) дает экономию25000 человеко-часов или 14 сотрудников персонала. По сравнению с затратами на разработку и поддержку фази системы эта экономия весьма существенна.
